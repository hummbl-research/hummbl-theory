# Contributing to hummbl-theory

Thank you for your interest in contributing to HUMMBL's theoretical foundations.

## How to Contribute

1. **Open an issue** describing the theoretical claim, correction, or addition you propose.
2. **Fork the repo** and create a branch: `feat/<short-desc>` or `fix/<short-desc>`.
3. **Follow evidence standards** -- every theoretical claim needs a citation, research note path, or explicit `[HYPOTHESIS]` / `[ESTIMATE]` tag.
4. **Submit a pull request** with a clear description of what changed and why.

## Evidence Standards

- All claims must cite a primary source (author, year, work).
- Unsourced claims must be tagged `[UNVERIFIED]`.
- Status tags are required: `[STATUS: LIVE]`, `[STATUS: DESIGNED]`, `[STATUS: PROPOSED]`, or `[STATUS: REJECTED]`.

## Terminology

- **Base120** -- the 120 mental-model operators (not "HCC")
- **VERUM** -- the append-only audit layer (not "Base4")
- **BKI** -- Belonging as Knowledge Infrastructure

## Code of Conduct

Be respectful, evidence-based, and intellectually honest. Ubuntu applies here too.
